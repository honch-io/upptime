# Honch Status — web

The Honch status page (https://status.honch.io). A Vite + React + Tailwind CSS v4
app built with **[coss ui](https://coss.com/ui)** components. It is a pure
read-only client over Upptime's committed data — it fetches `history/summary.json`,
the per-service history, `api/incidents.json`, and the perf snapshots at runtime
(no server, no external CDN at runtime; fonts are bundled locally).

## Develop

```bash
npm install
npm run dev        # local dev server
npm run build      # production build → dist/
npm run typecheck  # tsc --noEmit
```

## Deployment

`.github/workflows/deploy-status.yml` runs `npm ci && npm run build`, copies the
committed RSS/Atom feeds (generated by `feeds.yml`) into `dist/`, adds the CNAME,
and publishes `dist/` to the `gh-pages` branch.

## coss ui components

coss ui follows a "copy, paste, and own" model, so its component source lives in
the repo under `src/components/ui/` (`frame`, `button`, `badge`, `alert`,
`separator`, `skeleton`, `spinner`, `empty`, `dialog`), with the theme tokens in
`src/styles.css`. The **Frame** component (`Frame` / `FrameHeader` / `FrameTitle` /
`FrameDescription` / `FramePanel` / `FrameFooter`) is the primary building block:
the hero, KPI stats, per-service status list, performance sections, and incident
history are all composed as Frames. Runtime deps (`@base-ui/react`, `lucide-react`,
`class-variance-authority`, `clsx`, `tailwind-merge`) come from npm.
